A Note About TDF Files

TTDAlter 4.0 introduced the capability to save vehicle information in a small data file, called a TDF file. However, during the 4.0.x beta versions, new data was discovered and the TDF format had to change. In general, TDF files are forwards- and backwards-compatible within a minor version (eg, a 4.0.8 datafile will load into 4.0.9, and vice versa, but a 4.1.3 datafile will not load into 4.0.7), except in the following cases:

  • TTDAlter 4.0.4 files cannot be read by newer versions (4.0.4 was the first version of TTDAlter 4.0.x)
  • TTDAlter 4.0.5-4.0.8 datafiles are missing general data on vehicle life, introduction date, and climates, as well as data on disasters, as you could not alter disasters in these versions
  • TTDAlter 4.0.x datafiles had corrupt data on train cargo and double-headed values. Loading a TDF from a different version of TTD than yours (eg, loading a TDF created with the French DOS version into the to American Windows version) could actually corrupt your TTD EXE file!
  • TTDAlter 4.1 datafiles cannot be read by TTDAlter 4.0.x.

In general, you should always use the latest version of TTDAlter. If you have an old TDF file you want to read, but for some reason can't (for example, you created a TDF with TTDAlter 4.0.4, and want to use this with TTDAlter 4.1), e-mail me and I'll convert it to a 4.1 datafile.
