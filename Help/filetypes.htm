File Types

Transport Tycoon Deluxe uses the same format for saving games and scenarios. The only real difference is the file extension. This means that it's possible to load a scenario file into TTDX Editor and change many settings. However, not all options are available as no players are defined in scenarios.

These file extensions are used by the game:

Filetype Description
SS0 Built-in scenario
SS1 Built-in saved game
SV0 Player-created scenario
SV1 Normal saved game
SV2 Multiplayer saved game

Compressing and Uncompressing Saved Games

TTD compresses its data in a rather simple way. Unfortunately, TTD can only handle games saved in this way: files therefore have to be decompressed to be edited. TTDX Editor has options to save (and load) uncompressed files uncompressed as well as compressed files. This feature is targeted at people who wants to do further editing by hand. After editing, the file must then be loaded back in the editor and saved in the format known by TTD. If the filetypes have been associated with the editor both uncompressing and compressing options are included in the right-click menu in Explorer.

Files saved uncompressed use the original name but adds "hdr" and "dta" to the extension. An .sv1 (saved game) file becomes .sv1hdr and .sv1dta. The hdr file contains the header information for the game. This is just the display name of the game, there's usually nothing to change here. The dta file contains all the data from the actual savegame. Offsets listed in the Techinfo window point directly here.

Notes

The editor (just like the game) depends on the extensions for identification. In addition, the uncompressed size is also used for identification of the files.

There is a problem when using games and scenarios saved in the Windows version of TTD in the DOS version. This can be fixed by loading the file into this editor and then saving it again.

In versions 1.10.013 and earlier of TTDX Editor, you may have had trouble loading games saved with certain versions of TTDPatch. This was fixed in 1.10.014 (and a subsequent fix made in 1.10.015), and TTDX Editor can now load TTDPatch games fine.
